    },

Expand description

These asset variants can be loaded from configuration files. They will then replace a dynamic asset based on their keys.

Variants§

§

File

Fields

§path: String

Asset file path

A dynamic asset directly loaded from a single file

§

Folder

Fields

§path: String

Asset file folder path

A folder to load all including asset files from

Subdirectories are also included. This is not supported for web builds! If you need compatibility with web builds, consider using StandardDynamicAsset::Files instead.

Fields

§paths: Vec<String>

A list of files to be loaded as a vector of handles
