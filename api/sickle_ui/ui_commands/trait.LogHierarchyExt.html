fn log_hierarchy( &'a mut self, component_filter: Option<fn(_: ComponentInfo) -> bool> ) -> &mut EntityCommands<'a>

Logs the hierarchy of the entity along with the component of each entity in the tree. Components listed can be optionally filtered by supplying a component_filter

§Example
commands.entity(parent_id).log_hierarchy(Some(|info| {
    info.name().contains("Node")
}));
